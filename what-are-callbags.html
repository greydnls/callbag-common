linkWhat Are Callbags?

Take this code:

1linkconsole.log(funcA(...));

2linkconsole.log(funcB(...));

What happens here? First funcA() is called and the result it produces is logged, then funcB() is called and the result it produces is logged.


What if funcA() takes a long time to produce its result? Well instead of waiting for funcA() to finish and then calling funcB(), we could tell funcA() to take as long as it takes, but call us back when it has produced its result, and tend to funcB() in the meanwhile. I.e.:

1linkfuncA(..., (result) => console.log(result));

2linkconsole.log(funcB());


Now what if funcA() encounters an error while it is trying to produce its result? Well, the convention is that the callback we provide takes two arguments, and funcA() will send errors on the first and data on the second:

1linkfuncA(..., (err, result) => {

2link  if (err) { /* --> handle error */ }

3link  else console.log(result);

4link});

5linkconsole.log(funcB());

linkStreams

Now what if funcA() produces not one, but multiple results? For example, it might be a function who is supposed to calculate the position of the mouse cursor on the X-axis, or it might be the function who calculates the length of some live user input.

👉 We call a function that might produce an indeterminate number of results, at indeterminate points in time, a stream.


Well the classic callback spec is too limiting for that purpose, as a stream would need to say a few more things than just "Here is the data" or "I failed":

💬 "I will give you data when I have some. Tell me when you don't want anymore".

💬 "Here is some more data."

💬 "I am going to stop sending more data because of X."


With the classic callback spec, you are also more limited in your communication with the function / producer / data-source, as you can only say "I want data" by calling it. With a stream, however, you would also need more options:

🗨️ "Give me data when you got some."

🗨️ "Give me more data."

🗨️ "Stop giving data."

linkCallbags

We can group different messages that we want to send to / receive from streams like this:

GREETING
🗨️ "Give me data when you got some."
💬 "I will give you data when I have some. Tell me when you don't want anymore".

DATA
💬 "Here is some more data."
🗨️ "Give me more data."

END
💬 "I am going to stop sending more data because of X." /
🗨️ "Stop giving data."


Based on this, when we have a source stream, we need to be able to greet it, ask more data from it (perhaps optional based on what kind of source it is), or end it. When greeting it, we need to provide it a way to talk back to us, using which it can greet us, give us data, or tell us when it is going to stop.

We can model either direction of that communication with a function with two arguments:

  • One to specify the type of the message (GREETING, DATA, END)
  • One to specify the content of the message:
    • A function for talking back, in case of GREETING
    • The data being sent by the source, in case of DATA (can be empty when requesting data)
    • The reason for termination, in case of END (can be empty if no particular reason)

i.e.

1link(type: GREET | DATA | END, payload?: any) => void;

👉 This spec for callbacks is basically called the callbag specification. In other words, any function like that is a callbag.


Using this spec, if our funcA() is a stream, we provide it with a callbag instead of a classic callback:

1linkfuncA(GREET, (type, payload) => {

2link  if (type === DATA) console.log(payload);

3link  else { /* --> funcA() is finished, handle errors if necessary */ }

4link});

👉 In the callbag spec, message type is represented by numbers:

  • 0 means GREET
  • 1 means DATA
  • 2 means END

So the actual type definition for a callbag is like this:

1link(type: 0 | 1 | 2, payload?: any) => void;

linkTalking Back

Now what if we want to only receive 5 data points from funcA(), and end it afterwards? Well according to the spec we outlined, for every greeting message a way for talking back should be provided: another callbag. We can use that talkback to stop the source after we've got 5 data entries:

1linkfuncA(GREET, (type, payload) => {

2link  let talkback;

3link  let N = 0;

4link

5link  if (type === GREET) talkback = payload; // --> GREET message, payload is a callbag that allows us to talk back to the source

6link  else if (type === DATA) {

7link    console.log(payload);

8link    N++;

9link    if (N >= 5) talkback(END);            // --> We've got 5 data entries, ask the source to end.

10link  }

11link});


👉 In the examples so far, we have assumed that the source (funcA()) will give us data whenever it has some, without us requesting it. Sources like that are called listenable. If we need to ask a source for more data, i.e. we need to pull data from it, we call the source pullable.


Now what if funcA() is actually not a listenable source but a pullable source? Well, we would need to modify our code and use the talkback to also request data whenever we get some:

1linkfuncA(GREET, (type, payload) => {

2link  let talkback;

3link  let N = 0;

4link

5link  if (type === GREET) {

6link    talkback = payload;                   // --> GREET message, payload is a callbag that allows us to talk back to the source

7link    talkback(DATA);                       // --> pull data

8link  }

9link  else if (type === DATA) {

10link    console.log(payload);

11link    N++;

12link    if (N >= 5) talkback(END);            // --> We've got 5 data entries, ask the source to end.

13link    else talkback(DATA);                  // --> request more data

14link  }

15link});

linkCallbags in Practice

In practice, you rarely need to greet sources or handle talkbacks manually. Utilities such as those provided in callbag-common take care of all of that message passing for you:

1link​import { interval, pipe, map, filter, subscribe } from 'callbag-common'

2link​

3link​const source = interval(1000) // --> emits every second

4link​pipe(

5link​  source,

6link​  map(x => x * 3),            // --> multiply by 3

7link​  filter(x => x % 2),         // --> only allow odd numbers

8link​  subscribe(console.log)      // --> log any incoming number

9link​)
